## 🗄️ ثانياً: هندسة وتحديثات قاعدة البيانات (Database Architecture)

لتحقيق شرط المرونة؛ يعتمد النظام على مصفوفة **(Metadata-Driven Database)**؛ حيث تُخزن البيانات المشتركة في أعمدة منفصلة (لتسريع الفرز والبحث الجغرافي)، وتُخزن البيانات المتغيرة المخصصة في عمود واحد من نوع **JSON** يتم تفكيكه ومعالجته برمجياً. تم إدخال تعديلات هندسية هامة لتسريع جلب البيانات واستقرار الاستعلامات:

### 1. جدول السجلات الرئيسي (`records`)
يخزن المعاينات الميدانية؛ وتم تزويده بفهارس جغرافية وأعمدة استخلاص مخزنة لسرعة الاستعلام:
*   `geom` (POINT): حقل جيو-مكاني من نوع `POINT` يتم توليده وتحديثه تلقائياً بصيغة (Stored Virtual) بناءً على خطوط الطول والعرض المدخلة لتمكين الفهرسة الجغرافية السريعة:
    ```sql
    ALTER TABLE `records` ADD COLUMN `geom` POINT GENERATED ALWAYS AS (POINT(`longitude`, `latitude`)) STORED;
    ALTER TABLE `records` ADD SPATIAL INDEX `idx_spatial_geom` (`geom`);
    ```
*   `extracted_point_number` & `extracted_owner_name`: حقول افتراضية مخزنة مفهرسة تستخلص البيانات من الـ JSON تلقائياً لتسريع البحث النصي والجرد بصفحة التقارير:
    ```sql
    ALTER TABLE `records` ADD COLUMN `extracted_point_number` VARCHAR(50) GENERATED ALWAYS AS (json_unquote(json_extract(`dynamic_values`, '$.point_number'))) STORED;
    ALTER TABLE `records` ADD COLUMN `extracted_owner_name` VARCHAR(150) GENERATED ALWAYS AS (json_unquote(json_extract(`dynamic_values`, '$.owner_name'))) STORED;
    ALTER TABLE `records` ADD INDEX `idx_extracted_point_number` (`extracted_point_number`);
    ALTER TABLE `records` ADD INDEX `idx_extracted_owner_name` (`extracted_owner_name`);
    ```
*   `idx_records_type_id` & `idx_records_user_id`: فهارس مخصصة على حقول الربط لتسريع عمليات الـ JOIN.

### 2. جدول الحركة المستندية والصادر والوارد الجديد (`record_transfers`)
يخزن السجلات التاريخية لحركة السجلات والحيازة الإدارية بين الأقسام:
```sql
CREATE TABLE IF NOT EXISTS `record_transfers` (
  `id` INT(11) NOT NULL AUTO_INCREMENT,
  `record_ids` TEXT NOT NULL,                             -- أرقام المعاينات المشمولة بالتحويل (مثل "21,22,23")
  `sender_dept` VARCHAR(100) NOT NULL,                    -- الإدارة المرسلة (الصادر منها)
  `receiver_dept` VARCHAR(100) NOT NULL,                  -- الإدارة المستهدفة (الوارد إليها)
  `operator_id` INT(11) NOT NULL,                         -- الموظف المسؤول الذي أنشأ الحركة
  `status` ENUM('pending', 'approved') DEFAULT 'pending',    -- حالة المعاملة: pending (معلق)، approved (تم الاستلام)
  `notes` TEXT DEFAULT NULL,                              -- ملاحظات تسليم العهدة
  `created_at` TIMESTAMP NULL DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  KEY `operator_id` (`operator_id`),
  CONSTRAINT `record_transfers_ibfk_1` FOREIGN KEY (`operator_id`) REFERENCES `users` (`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

### 3. جدول أنشطة النظام الرقابي (`audit_logs`)
تم تزويده بفهارس مخصصة لـ `user_id` و `created_at` لتسريع تحميل لوحة الرقابة التفاعلية وتصفيتها بالثانية.

## 💻 ثالثاً: التفصيل التشغيلي للموديولات والوظائف المطورة (Modules Walkthrough)

### 1. شريط وجرس الإشعارات المطور والقائمة المنسدلة للمستخدم (`index.php`)
تم تطوير الشريط العلوي لنقل جرس الإشعارات والقائمة المنسدلة لملف الموظف إليه مباشرة في الجهة اليسرى:
*   **استعلامات النشاط الفورية:** يجلب السيرفر آلياً إجمالي السجلات، والمدخلات اليومية، والمدخلات الشهرية، وتفاصيل آخر معاينة مدخلة واسم كاتبها من قاعدة البيانات.
*   **لوحة الإشعارات المنسدلة:** تظهر نقطة حمراء متحركة ومتحركة (`animate-bounce`) فوق الجرس عند رصد مدخلات جديدة اليوم؛ وعند النقر على الجرس تنسدل لوحة إحصائية أنيقة تعتمد على الألوان الداكنة لعرض ملخص فوري لنشاط المنصة اليومي.
*   **قائمة الموظف المنسدلة:** تم دمج اسم الموظف ورتبته في قائمة منسدلة منبثقة تفاعلية تتيح الانتقال السريع لـ (الملف الشخصي) أو (تسجيل الخروج الآمن).

### 2. معالج الاستيراد وتوطين القوالب المعتمدة (`import-view.php`)
تم فصله كلياً ليعمل بأداء فائق الخفة، ويحتوي على مزايا نوعية:
*   **منع التكرار ثنائي الحقول (Dual-Field Deduplication):** يتيح للمشرف تحديد حقلين وقائيين من حقول النظام (مثل رقم النقطة ورقم المعاملة) للتحقق المتبادل منهما. يقوم السيرفر بفحص حقل الـ JSON مباشرة عبر دالة `JSON_UNQUOTE(JSON_EXTRACT())`؛ وإذا وجد تطابقاً لأي منهما يتم تخطي الصف المكرر وتوثيقه في عداد السجلات المتخطاة.
*   **توليد قوالب إكسل مدعمة بالـ Data Validation:** عند تنزيل قالب الاستيراد المخصص للقسم، يقوم السيرفر آلياً بتحويل أرقام حقول قاعدة البيانات إلى حروف الأعمدة المقابلة لها في إكسل، ويحقن تاجات المواءمة القياسية لمايكروسوفت إكسل (`<x:DataValidation>`) بداخل مخطط ملف الـ XLS المولد باستخدام الفاصلة المنقوطة (`;`) المتوافقة مع إكسل والويندوز العربي. هذا يظهر خيارات القوائم المنسدلة والـ Checkbox (نعم/لا) تلقائياً داخل خلايا شيت الإكسل لدى الموظف في الميدان لضمان دقة الإدخال.
*   **معالجة تواريخ إكسل الرقمية (Excel Serial Dates):** يشتمل الملف في أعلاه على دالة حاسبة ذكية تقوم تلقائياً بتحويل الأرقام التعاقبية لملفات إكسل (مثل `46023`) إلى تاريخ مقروء وصريح بالصيغة القياسية (`YYYY-MM-DD`) قبل حفظها في الـ JSON لمنع تشوه التقارير.
*   **محرك المطابقة اللغوية والبرمجية التلقائي (Semantic Mapping Engine):** بمجرد اختيار القسم، يقوم كود جافا سكريبت بمطابقة أسماء أعمدة ملفك المرفوع مع أسماء حقول النظام تلقائياً وتحديد المواءمة، لتضيء الكروت المطابقة تلقائياً باللون الأخضر الجميل وظهور علامة صح نشطة ومتحركة (`fa-circle-check`).
*   **شيت التدقيق والتحقق الذكي الفوري (Smart Auditing Spreadsheet):** يوفر جدول مراجعة يدوي مساعد منسق بالكامل يعرض كافة الصفوف والمدخلات كحقول نصية وخيارات منسدلة مخصصة مطابقة تماماً لنوع الحقول المسجلة بالنظام لتعديلها وحفظها على الشاشة مباشرة. تم تزويده بـ (مكتشف ومصحح الأخطاء التلقائي)؛ حيث يقوم بفحص الخلايا ومطابقتها وتلوينها باللون الأحمر مع شارة "مطلوب" أو البرتقالي مع شارة "غير صالح للإحداثيات" عند وجود أي خطأ، مع شريط حالة تفاعلي علوي يعطل زر الحفظ حتى يتم تصحيح الأخطاء.

### 3. مركز الصادر والوارد وتتبع الحركة المستندية (`transfers-view.php`)
موديول تفاعلي جديد بالكامل ينظم سلسلة الحيازة للسجلات بين الإدارات:
*   **التصدير وصنع الصادر:** يتيح للمشرف تحديد عدة سجلات في لوحة الإدارة وتصديرها كشحنة واحدة وتحديد الإدارة المستلمة وملاحظات التسليم، لتتغير حالة السجلات تلقائياً بالنظام إلى: **"صادر معلق إلى [الجهة المستلمة]"**.
*   **التأكيد والاستلام التلقائي للوارد:** عند دخول موظف الجهة المستلمة، تظهر له تنبيهات الشحنات المعلقة. بمجرد ضغطه على زر "تأكيد الاستلام"؛ يتم تحديث حالة السجلات تلقائياً بجدول الـ JSON لتصبح: **"مستلمة وفي حوزة [الإدارة الجديدة]"** وتوثق العملية في سجل الرقابة بالتوقيت والموظف المستلم بدقة تامة.
*   **البحث والجرد التاريخي:** تم تزويد الموديول بمحرك بحث متطور وجدول مؤرشف يتيح للمدير الفرز ومعرفة كافة السجلات التي نُقلت لإدارة معينة خلال فترة زمنية محددة بدقة.

### 4. لوحة جرد وإدارة السجلات الميدانية المحدثة (`records-manage.php`)
*   **التقسيم البرمجي لصفحات (Server-Side Pagination):** يعرض 20 سجلاً فقط بالصفحة من قاعدة البيانات لضمان الخفة والسرعة العالية بالتحميل، مع شريط تنقل سفلي أنيق يحافظ على فلاتر البحث الحالية أثناء التنقل.
*   **مربعات التحديد والعمليات الجماعية للصادر والوارد:** تم دمج مربعات التحديد (Checkboxes) بجانب كل صف مع شريط عمليات عائم سفلي يظهر عند التحديد لتصدير المعاملات المحددة كشحنة صادر دفعة واحدة.

### 5. لوحة المؤشرات البيانية وباني الـ KPIs المطور (`dashboard-view.php`)
*   **الرسوم البيانية الأساسية المصفاة:** رسوم دائرية لتوزيع السجلات بالأقسام ورسوم خطية لمعدل النمو الشهري تستقبل الفلاتر الزمنية والفنية وتتحدث آلياً.
*   **باني ومصمم الكروت والمؤشرات المخصصة (KPIs):** موديول مستقل يتيح للمشرف تحديد الحقول الفنية المُراد إبراز كروت وإحصائيات تكرارها، ليعرض كرت موحد لكل حقل يحتوي بداخله على الأعداد الإجمالية أو تكرار عناصر القائمة المنسدلة فقط بألوان زاهية وخطوط عريضة مريحة للعين مع دعم طباعة تقارير المؤشرات لـ PDF.

### 6. الخريطة التفاعلية ورسم الحدود الإدارية المباشر (`map-view.php`)
*   **تكتيل وتجميع الأيقونات (Marker Clustering):** مدمج بها نظام تجميع ذكي لضغط آلاف النقاط في دوائر متوهجة ديناميكياً لتسريع استجابة الخريطة، مع ميزة الانتشار العنكبوتي عند تطابق الإحداثيات والتقريب التلقائي، وإلغاء التكتيل عند التقريب لمستوى الشارع.
*   **باني ومخطط الحدود الإدارية المباشر (Leaflet Geoman):** تم دمج مكتبة رسم خفيفة ومعربة بالكامل تظهر لمدير النظام (Admin) فقط. عند انتهاء المدير من رسم أي مضلع جغرافي على الخريطة، يتم استدعاء دالة `generateKmlFromLatLngs` التي تقوم بالتقاط النقاط الجغرافية وصياغة مضلع KML مغلق وسليم هندسياً (مرتب بتقديم خط الطول أولاً ثم العرض ليتوافق مع معايير جوجل إيرث). ثم يتم ترميز كود الـ KML المولد بصيغة **`Base64`** وإرساله عبر طلب POST آمن إلى السيرفر لحفظه بجدول `boundaries` وتنشيطه فوراً.

### 7. لوحة الرقابة وسجل العمليات الأمنية التفاعلية (`audit-logs.php`)
تم تطويرها لتصبح مركز رقابة أمني متكامل:
*   **تصنيف وتلوين الإجراءات تلقائياً:** يتم فرز وتلوين العمليات آلياً بناءً على الأهمية الأمنية ودرجة الخطورة (اللون الأحمر ورمز الإنذار `🚨` لعمليات الحذف والتصفير والتطهير الحساسة، واللون البرتقالي لعمليات التحديث والتعديل، والأخضر للإضافات والإنشاء الميداني، والأزرق لعمليات الدخول وتغيير الإعدادات).
*   **البحث الفوري المباشر (Live Search):** تم دمج شريط بحث تفاعلي يعمل بالـ JS لفلترة وجرد السجلات المعروضة في الجدول في نفس اللحظة دون الحاجة لانتظار تحميل الصفحة.
*   **تصدير التقارير الرقابية لـ PDF:** زر مخصص لتوليد وتصدير تقرير الرقابة الأمنية كملف PDF نظيف ومنسق بالكامل لعرضه على الجهات الإدارية الرقابية المسؤولة.

### 8. صيانة وتطهير السجلات وقاعدة البيانات (`backup-view.php`)
*   **الصيانة التلقائية وتطهير الملفات اليتيمة:** تقوم أداة الصيانة الذكية المحدثة بجرد ومطابقة مجلدات المرفقات الفعلية بالسيرفر ومقارنتها بقاعدة البيانات وحذف أي ملف يتيم أو صور ميتة غير مسجلة لتوفير مساحة السيرفر فورياً.
*   **أدوات التطهير والحذف المخصصة للسجلات:** تم دمج ميزة الحذف والتطهير المتقدمة المخصصة، والتي تمكن الإدارة من:
    *   حذف سجلات فترة زمنية محددة.
    *   حذف سجلات قسم ميداني معين.
    *   تصفير وإفراغ كامل السجلات الميدانية بالكامل (Truncate).
    *   *صمام الأمان التفاعلي لحفظ السجلات:* تم تأمين هذه العمليات الحساسة بطلب كتابة كلمة تأكيدية صريحة وهي كلمة **`DELETE`** أو **`حذف`** في شاشة منبثقة تفاعلية قبل تفعيل زر الحذف لتجنب أي حذف خاطئ.

### 9. القائمة الجانبية المنزلقة المتجاوبة للهواتف (`includes/sidebar.php`)
تمت إعادة تصميم السايدبار بالكامل ليتلاءم مع شاشات الهواتف الذكية:
*   **نظام الدرج المنزلق (Responsive Drawer Sidebar):** في شاشات الهاتف المحمول يتم إخفاء السايدبار بالكامل خارج الشاشة يميناً (`translate-x-full`)؛ وعند النقر على زر التحكم العلوي ينسلق السايدبار كـ درج جانبي فخم يغطي الشاشة مع ظهور خلفية ضبابية مظلمة (Overlay Background) عند النقر عليها يغلق السايدبار تلقائياً، دون مضايقة أو حجب الخريطة وموقع الـ GPS الفعلي.
*   **فرز صلاحيات الموديولات الجديدة:** تم ربط السايدبار ديناميكياً لفرز الأزرار الجديدة (الصادر والوارد، الاستيراد، التصدير) وعرضها بناء على تصاريح الموظف النشطة.

## 🔒 رابعاً: التدابير والحماية الأمنية للنظام (Security Architecture)

تم تأسيس البنية الأمنية للمنصة لمنع محاولات الاختراق، حقن البيانات، أو تسريب تفاصيل السيرفر:

1.  **تجاوز اعتراض جدران الحماية الصارمة (WAF Bypass):**
    لحماية مدخلات النظام وتفادي حظر جدران الحماية الخبيث (ModSecurity 403)؛ تم استخدام تقنية **ترميز الـ Base64 لبيانات الاستمارات الحساسة** في طرف العميل (المتصفح) قبل إرسالها (مثل حقول تشييد الإعدادات ونصوص الـ KML المفتوحة للحدود المرسومة يدوياً)، ومن ثم فك ترميزها برمجياً عبر دالة `base64_decode` داخل السيرفر قبل معالجتها وحفظها.
2.  **تأمين وحماية ملفات الكوكيز وجلسات المشرفين (Session Security):**
    تم ضبط معايير أمان ملفات تعريف الارتباط تلقائياً في أعلى ملف `config.php` قبل استدعاء `session_start()` لضمان حماية الجلسات ضد ثغرات الـ XSS وخطف الكوكيز برمجياً عبر تفعيل خيار **`httponly`**، وتفعيل خيار **`samesite => Lax`** لحظر إرسال الكوكيز من الروابط الخارجية، وتشفير حركة البيانات تلقائياً في حال توفر بروتوكول الـ HTTPS الآمن عبر خيار **`secure`**.
3.  **منع تسريب تفاصيل الاتصال البرمجية (Information Disclosure Prevention):**
    تم تأمين كتلة اصطياد أخطاء الاتصال بـ PDO داخل ملف الاتصال `db.php`؛ حيث تم إيقاف طباعة الأخطاء الصريحة للمسؤول وتوجيه الخطأ التفصيلي الحقيقي إلى ملف سجلات الأيرور المغلق بالسيرفر برمجياً عبر دالة `error_log()` لتجنب تسريب اسم مستخدم قاعدة البيانات أو المسارات الداخلية للمنصة.
4.  **منع تداخل وتراكم كاش الهواتف (Passive PWA Service Worker):**
    تمت إعادة هندسة وصياغة ملف الخدمة `sw.js` ليكون خاملاً ومقتصراً على تأمين شرط تثبيت التطبيق على الشاشات، مع برمجته ليقوم **بتدمير وتطهير كاش المتصفحات القديمة للمستخدمين تلقائياً**، لمنع حجب الصلاحيات وتحديث السايدبار وعدادات الإشعارات في صفحات النظام بشكل فوري من السيرفر.
